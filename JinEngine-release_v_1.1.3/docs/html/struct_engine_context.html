Detailed Description

Aggregates pointers to core engine subsystems.

EngineContext acts as a lightweight dependency container that is passed throughout the engine during update, render, and lifecycle calls.

It provides unified access to major engine systems such as:

  • State management
  • Window and input handling
  • Rendering
  • Audio
  • Core engine control

EngineContext does not own these systems. It only stores raw pointers that are managed and lifetime-controlled by JinEngine.

This structure is intentionally simple and copyable, and is commonly passed by const reference to avoid tight coupling between systems.
