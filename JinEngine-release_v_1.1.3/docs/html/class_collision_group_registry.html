Detailed Description

Registry that assigns bit indices to collision group tags.

CollisionGroupRegistry maintains a stable mapping between string tags (e.g., "Player", "Enemy", "Pickup") and a unique bit index.

The registry supports:

  • Allocating a new bit when a previously unseen tag is requested
  • Reverse lookup from bit index back to tag for debugging/inspection

This is typically used to build collision filtering masks (group membership and interaction rules).
