Detailed Description

Utility structure for frame timing and FPS calculation.

EngineTimer provides basic timing functionality used by the engine loop. It measures delta time between frames and optionally computes frames-per-second (FPS) over a fixed accumulation interval.

The timer relies on time information provided through EngineContext (typically sourced from a high-resolution platform timer).

This structure is intended to be owned and driven by the core engine loop.

bool EngineTimer::ShouldUpdateFPS ( float & outFPS)
nodiscard

Determines whether an updated FPS value should be reported.

This function accumulates frame count and elapsed time internally. When a predefined time interval has elapsed (commonly 1 second), it computes the average FPS, writes it to outFPS, resets the internal FPS counters, and returns true.

If the interval has not yet elapsed, the function returns false and outFPS remains unmodified.

Parameters
outFPSOutput parameter that receives the calculated FPS value.
Returns
true if a new FPS value was computed; false otherwise.

◆ Tick()

float EngineTimer::Tick ( const EngineContext & engineContext)
nodiscard

Advances the timer and computes delta time for the current frame.

This function retrieves the current time from EngineContext, computes the elapsed time since the previous frame, updates internal accumulators, and returns delta time in seconds.

The returned delta time is typically used to drive frame-based updates throughout the engine.

Parameters
engineContextEngine context providing the current time source.
