\doxysubsection{Detailed Description}
Compute-\/only material that binds textures as images for compute shader dispatch. 

\doxylink{class_compute_material}{Compute\+Material} specializes \doxylink{class_material}{Material} for compute workloads\+:
\begin{DoxyItemize}
\item \doxylink{class_compute_material_ad5a9a88431fa1b207fb59f4d52528fe2}{Is\+Compute()} returns true so \doxylink{class_render_manager}{Render\+Manager} can route it through Dispatch\+Compute().
\item \doxylink{class_compute_material_a50f183754e1a0235cb0a061aef1ef72f}{Set\+Image()} binds textures as images (\doxylink{class_texture_a28b89a5f6f78de411c85fc83c04963b9}{Texture\+::\+Bind\+As\+Image}) instead of sampler textures.
\item destination\+Texture is tracked for dispatch dimension calculation in \doxylink{class_render_manager}{Render\+Manager} (e.\+g., using output size).
\end{DoxyItemize}

Restrictions\+:
\begin{DoxyItemize}
\item Instancing is not applicable for compute, so Enable\+Instancing is deleted.
\item Sampler-\/based Set\+Texture is deleted to prevent accidental use; compute uses images instead.
\end{DoxyItemize}

Convention\+:
\begin{DoxyItemize}
\item If \doxylink{class_compute_material_a50f183754e1a0235cb0a061aef1ef72f}{Set\+Image()} is called with uniform\+Name == "{}u\+\_\+\+Dst"{}, destination\+Texture is updated to that texture. \doxylink{class_render_manager}{Render\+Manager} typically uses this to compute dispatch group counts. 
\end{DoxyItemize}
