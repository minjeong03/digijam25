\doxysubsection{Detailed Description}
Central rendering system responsible for resource registration and batched object rendering. 

\doxylink{class_render_manager}{Render\+Manager} serves two primary roles\+: 1) Resource registry (tag -\/\texorpdfstring{$>$}{>} Shader/\+Texture/\+Mesh/\+Material/\+Font/\+Sprite\+Sheet). 2) Per-\/frame rendering pipeline for Objects, including visibility culling, render sorting, batching, and drawing.

Rendering flow (driven by \doxylink{class_jin_engine}{Jin\+Engine} / \doxylink{class_state_manager}{State\+Manager} / \doxylink{class_object_manager}{Object\+Manager})\+:
\begin{DoxyItemize}
\item Begin\+Frame(engine\+Context)\+: optionally binds an offscreen framebuffer (scene\+FBO) and clears it.
\item Submit(objects, engine\+Context)\+: frustum-\/culls objects using the active camera and builds a \doxylink{_render_manager_8h_ac442dd962bc57bfa60125f977d6edea5}{Render\+Map}.
\item Flush\+Draw\+Commands(engine\+Context)\+: iterates \doxylink{_render_manager_8h_ac442dd962bc57bfa60125f977d6edea5}{Render\+Map} and issues draw calls (instanced when possible).
\item Flush\+Debug\+Line\+Draw\+Commands(engine\+Context)\+: draws accumulated debug line geometry.
\item End\+Frame(engine\+Context)\+: if offscreen is enabled, blits the scene target to the default framebuffer.
\end{DoxyItemize}

Offscreen rendering\+:
\begin{DoxyItemize}
\item When use\+Offscreen is true, the manager renders into scene\+FBO with a floating-\/point color target (GL\+\_\+\+RGBA16F).
\item The scene target is recreated when the window size changes (Begin\+Frame and On\+Resize).
\item The offscreen color texture is force-\/registered under tag "{}\mbox{[}\+Engine\+Texture\mbox{]}\+Render\+Texture"{}.
\end{DoxyItemize}

Batching \& sorting\+:
\begin{DoxyItemize}
\item \doxylink{class_render_manager_a8e033cb685d2619f9b5dfec8a165e387}{Build\+Render\+Map()} groups visible objects into \doxylink{_render_manager_8h_ac442dd962bc57bfa60125f977d6edea5}{Render\+Map}\mbox{[}layer\mbox{]}\mbox{[}zbin\mbox{]}\mbox{[}shader\mbox{]}\mbox{[}\doxylink{struct_instance_batch_key}{Instance\+Batch\+Key}\mbox{]} buckets.
\item depth is quantized into an integer bin (depth \texorpdfstring{$\ast$}{*} 1000, rounded) to provide stable depth ordering while still using an ordered map for deterministic iteration.
\end{DoxyItemize}

Instancing\+:
\begin{DoxyItemize}
\item If the first object in a batch reports Can\+Be\+Instanced(), the entire batch is drawn using \doxylink{class_mesh_ae2933e498cc249ab798bbf265dace77b}{Mesh\+::\+Draw\+Instanced()}.
\item Per-\/instance data is built on CPU (model matrices, colors, UV offset/scale) and uploaded via \doxylink{class_mesh_a45673d65011bf8f70c0f7de2a37d7f6c}{Mesh\+::\+Update\+Instance\+Buffer()}.
\end{DoxyItemize}

Safety for unregistering\+:
\begin{DoxyItemize}
\item Unregister\texorpdfstring{$\ast$}{*} functions validate that no live \doxylink{class_object}{Object} references the target resource before erasing it.
\item If references exist, the resource is not deleted and a warning is logged. 
\end{DoxyItemize}
